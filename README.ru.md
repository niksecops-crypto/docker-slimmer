# Docker-Slimmer: Профессиональный оптимизатор Docker-образов

Docker-Slimmer — это автоматизированный инструмент, предназначенный для преобразования устаревших, тяжеловесных Docker-файлов в высокоэффективные, безопасные и легковесные многоэтапные сборки (multi-stage builds). Инструмент помогает DevOps-инженерам модернизировать унаследованные контейнеры, применяя лучшие практики индустрии: использование Distroless образов и агрессивную очистку кэша.

## Ключевые возможности

- **Автоматизация Multi-Stage сборок**: Автоматическое разделение процесса на этапы сборки и рантайма.
- **Поддержка Distroless**: Использование минимальных базовых образов Google Distroless для сокращения поверхности атаки и минимизации размера.
- **Автоматическая очистка кэша**: Встроенная поддержка менеджеров пакетов `apt` и `apk` для удаления временных файлов после установки зависимостей.
- **Security-First подход**: Запуск от пользователя `nobody` по умолчанию и удаление ненужных системных утилит (shell и т.д.).
- **Изоляция артефактов**: Копирование только необходимых бинарных файлов и конфигураций в финальный образ.

## Почему стоит использовать Docker-Slimmer?

Унаследованные (legacy) Docker-образы часто содержат более 500 МБ лишних данных (компиляторы, инструменты сборки, кэши пакетов). Docker-Slimmer может сократить размер образа до 90%, значительно повышая безопасность за счет удаления доступа к оболочке (shell) и устранения уязвимостей (CVE), присутствующих в базовых дистрибутивах ОС.

## С чего начать?

### Требования

- Go 1.22+ (для сборки инструмента)
- Docker (для сборки оптимизированных образов)

### Установка

```bash
git clone https://github.com/niksecops-crypto/docker-slimmer.git
cd docker-slimmer
go build -o slimmer ./cmd/slimmer
```

### Примеры использования

Оптимизация старого Go-образа путем извлечения бинарного файла в Distroless рантайм:

```bash
./slimmer generate --base golang:1.22-alpine --artifacts /app/main --output Dockerfile.optimized
```

## Рекомендации для продакшена

- **Сканирование до и после**: Используйте `trivy` или `grype`, чтобы убедиться в сокращении количества уязвимостей.
- **Тестирование Entrypoints**: Убедитесь, что все runtime-зависимости явно скопированы в финальный этап.
- **Distroless для безопасности**: Используйте `gcr.io/distroless/static` для статически скомпилированных приложений (Go, Rust).
